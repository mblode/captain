Captain a fleet of [cmux](https://cmux.com/) worktrees from one Claude Code session, one worktree per Linear ticket.

Each worktree runs its own agent on a brief covering the whole job: plan, implement, review, open a PR. From your captain session you watch them with `captain status`, approve their plans, and answer what they're blocked on.

## Requirements

- Node.js >= 22
- `git`, `claude` ([Claude Code](https://claude.com/claude-code)), and [`cmux`](https://cmux.com/) on your PATH
- `LINEAR_API_KEY` (optional, pulls ticket details and screenshots into each brief)

## Install

```bash
npm i -g cmux-captain   # the CLI
captain install         # add the skills the fleet needs, then check your setup
```

`captain install` adds the `/captain` skill you drive plus the pipeline skills each brief invokes (`/pr-reviewer`, `/pr-creator`, `/pr-babysitter`), then verifies node, git, claude, cmux, and `LINEAR_API_KEY`. Re-run it any time to re-check.

See [Usage](/usage) for the commands and how a worktree reaches "ready to merge".
